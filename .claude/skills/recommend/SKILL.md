# AI Trend Recommend Skill

このリポジトリの `trend/YYYY/MM/YYYYMMDD.md` を読み込み、AI評価を付与した `YYYYMMDD-ai-recommend.md` を生成する。

## 引数の解釈

スキル呼び出し時の引数を以下のルールで解釈する:

- **日付指定** (`YYYYMMDD`、8桁): その日のファイル1件を処理
- **月指定** (`YYYYMM`、6桁): その月の全 `YYYYMMDD.md` ファイルを処理（既存の `-ai-recommend.md` はスキップ）
- **引数なし**: `trend/` 配下で最新の `YYYYMMDD.md`（対応する `-ai-recommend.md` が存在しないもの）を1件処理

## 実行手順

### Step 1: 対象ファイルの特定

**日付指定の場合:**
```
入力ファイル: trend/YYYY/MM/YYYYMMDD.md
出力ファイル: trend/YYYY/MM/YYYYMMDD-ai-recommend.md
```
例: `20260624` → `trend/2026/06/20260624.md`

**月指定の場合:**
```bash
# 対象月のYYYYMMDD.mdを列挙し、-ai-recommend.mdが存在しないものを処理
find trend/YYYY/MM/ -name "????????.md" | sort
```

**引数なしの場合:**
```bash
# 最新の未処理ファイルを探す
find trend/ -name "????????.md" | sort | tail -n 20
# 対応する -ai-recommend.md がないものの中で最新を選ぶ
```

### Step 2: 入力ファイルの読み込み

入力ファイルを読み込み、以下を確認する:
- セクション（ソース名の見出し）の一覧
- 各セクションの「注目トピック」テーブルの列構成（ブクマ数列があるかどうかはソースにより異なる）
- 各セクションの「全エントリー」リスト（評価対象記事の正式な一覧）

### Step 3: 記事の評価

「全エントリー」から重複URLを除去した記事リストを作成し、各記事に以下を付与する:

**scoreの基準:**
- `3`: AI・backend・frontend・infra・architecture・devtools に直接関係するソフトウェアエンジニアリングの記事
- `2`: 技術トレンド・エンジニアリング文化に間接的に関係する記事
- `1`: 一般的なIT/テックニュース
- `0`: 技術に無関係な記事

**categoryの選択肢（必ずこの中から1つ）:**
`backend` / `frontend` / `infra` / `ai` / `architecture` / `devtools` / `other`

**興味度の変換:**
- score 3 → `★★★`
- score 2 → `★★`
- score 1 → `★`
- score 0 → `-`

**memoのルール:**
- 必ず日本語で2〜3文
- 記事内容の要約と技術的な関連性を簡潔に述べる

### Step 4: 出力Markdownの生成

**必ず守るルール:**
- 入力ファイルの見出し・セクション・テーブル構造を完全に維持する
- 「注目トピック」テーブルの `興味度 | カテゴリ | メモ` 列のみ埋める
- タイトル、URL、既存の数値列（ブクマ数・いいね数・スコア等）は変更しない
- 「全エントリー」リストはそのまま保持する
- 入力にない記事を追加しない
- メトリクス列が存在しないテーブルには追加しない

**出力行の例:**
```md
| [記事タイトル](https://example.com) | 120 users | ★★★ | ai | LLMエージェント開発の実装知見が具体的に示されています。実運用で再現しやすい手順があり、優先度が高いです。 |
```

メトリクス列なしの例:
```md
| [記事タイトル](https://example.com) | ★★ | backend | Rustの採用事例とパフォーマンス改善の実測値が示されています。バックエンド開発者に参考になる内容です。 |
```

### Step 5: ファイル保存と報告

出力ファイルを `trend/YYYY/MM/YYYYMMDD-ai-recommend.md` に保存する。

月単位処理の場合は対象ファイルを順番に処理し、すべて完了したら処理結果の一覧を報告する。

最後に生成した出力ファイルのパスを報告する。

## 注意事項

- `興味度` 列には必ず `★★★` / `★★` / `★` / `-` の記号を使用する（数値不可）
- カテゴリは許可リスト外の値を使用しない
- 既存の `-ai-recommend.md` が存在する場合は上書きを確認するか、月単位処理ではスキップする
